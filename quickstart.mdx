Objective: working end to end in under 10 minutes.

## Prerequisites

- API key
- A folder of images

## Step 1: Create a batch

Use multipart form data to send images and optional metadata.

```bash
curl -X POST https://api.yourdomain.com/v1/batches \\
  -H "Authorization: Bearer $API_KEY" \\
  -F "files[]=@images/site-1.jpg" \\
  -F "files[]=@images/site-2.jpg" \\
  -F 'metadata={"site":"field-a","date":"2024-05-01"}' \\
  -F 'options={"output_formats":["csv"],"include_explanations":true}'
```

**Recommended limits**
- Max images per batch: 500
- Max file size per image: 25 MB
- Types: jpg, jpeg, png

The response is `202 Accepted` and returns `batch_id`.

## Step 2: Poll for status

```bash
curl -X GET https://api.yourdomain.com/v1/batches/$BATCH_ID \\
  -H "Authorization: Bearer $API_KEY"
```

Terminal states: `succeeded`, `failed`, `cancelled`. Poll every 2â€“5 seconds for small batches and back off if you receive 429.

## Step 3: Download results

```bash
curl -X GET "https://api.yourdomain.com/v1/batches/$BATCH_ID/results?format=csv" \\
  -H "Authorization: Bearer $API_KEY" \\
  -o results.csv
```

Use `format=xlsx` or `format=json` to change the output.

## Expected output (CSV sample)

```csv
image_id,filename,problem_likely,severity,ignore,confidence
img_01,site-1.jpg,true,high,false,0.92
img_02,site-2.jpg,false,none,true,0.81
img_03,site-3.jpg,true,medium,false,0.67
```

## Common quickstart pitfalls

- 401: missing or invalid key
- 413: request too large
- 429: rate limited; slow down polling or creation rate
